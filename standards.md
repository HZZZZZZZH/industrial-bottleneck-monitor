# 工业瓶颈投研系统标准

## 1. 入池标准

一只股票能进入观察池，必须同时满足下面四个条件。

### 1.1 行业范围

必须属于工业、硬科技、资源、能源、交通或国防等供需错配行业。

优先方向：

- AI 基础设施
- 电力设备
- 光子学 / 光通信
- 先进封装
- 半导体材料与设备
- 军工与航天
- 矿业与关键资源
- 航运、物流、能源运输

不纳入：

- 纯营销概念
- 没有硬资产、产能、认证、订单或技术节点的主题股
- 只能靠情绪传播、无法被季度数据追踪的标的

### 1.2 可追踪指标

必须至少存在一个资本市场能持续追踪的指标。

可接受指标：

- 订单
- backlog
- pipeline
- 产能
- 出货量
- 价格 / ASP
- 毛利率
- 客户认证
- design-in / qualification
- 政策许可
- 融资与扩产进度
- 第三方产业数据

核心原则：

如果指标不能被持续追踪，就不能进入系统。

### 1.3 时代 Beta

必须处在一个足够大的时代浪潮中。

示例：

- AI 训练 / 推理基础设施
- 数据中心电力重构
- 铜互连向光互连演进
- HBM / 先进封装扩产
- 工业电气化
- 国防补库存
- 关键矿产供给重估

不是所有增长都值得入池。只有当需求来自大周期，而不是单一客户短期订单，才符合系统逻辑。

### 1.4 外部验证信号

不能只是主观故事，至少要有一个外部验证信号。

外部验证可以很早期，但必须真实存在。

强验证：

- 财报收入增长
- backlog 增长
- 正式订单
- 长期供货协议
- 客户量产

中验证：

- pipeline 快速增长
- 客户认证
- design-in
- 设备订单
- 政策补贴
- 产能扩建

弱验证：

- 第三方产业报告确认技术路线
- 头部客户公开讨论架构变化
- 供应链玩家开始扩产
- 行业会议密集展示相关方案

弱验证可以入池，但只能进入前瞻池，不能直接视为主仓候选。

## 2. 架构演进预判

系统必须允许早期技术路线入池。

比如 InP 衬底、CPO、外部光源、光互连，在早期阶段可能还没有大规模收入，甚至市场普遍认为替代需要 2-3 年。但如果它处在下一代架构演进的关键位置，并且已经出现外部验证信号，就可以进入观察池。

判断问题：

- 当前架构的瓶颈是什么？
- 下一代架构是否必须绕过这个瓶颈？
- 新架构是否需要新的材料、设备、器件或产能？
- 供应链中谁最难扩产？
- 有哪些领先指标能证明架构切换正在发生？

如果回答不清楚，只能当主题观察，不能入池。

## 3. 入池分层

### 前瞻池

条件：

- 架构演进逻辑强
- 外部验证仍弱
- 财报尚未体现

典型指标：

- pipeline
- design-in
- 产业会议
- 设备订单
- 客户试产

仓位含义：

只能作为期权型观察，不作为核心仓。

### 验证池

条件：

- 已有订单、backlog、客户认证或产能扩建
- 需求正在从故事进入商业化

典型指标：

- backlog 增长
- 订单公告
- 产能 sold out
- 毛利改善
- 客户认证完成

仓位含义：

可以跟踪或小仓配置。

### 兑现池

条件：

- 收入、毛利、指引已经验证
- 市场仍低估后续连续上修

典型指标：

- revenue beat
- guidance raise
- 毛利率上行
- 经营现金流改善
- 客户追加订单

仓位含义：

可以成为主仓候选，但必须持续监控拥挤度和需求验证后涨幅。

## 4. 需求验证后涨幅

系统不使用“主观加入观察池”作为涨幅起点。

涨幅起点必须来自投资逻辑中的需求验证事件：从“仅叙事”进入“订单线索”开始计算。

定义：

```text
需求验证日 = 首次出现订单线索、backlog、客户认证、正式 pipeline 转化或财报验证的日期
验证价格 = 需求验证日收盘价
验证后涨幅 = 当前价 / 验证价格 - 1
```

系统展示时必须同时显示：

- 验证入池日
- 验证触发事件
- 验证价格
- 验证后涨幅

如果无法说清楚“几月几号因为什么信号入池”，则这只股票只能放在待核验池，不能使用验证后涨幅作为拥挤度指标。

档位：

- 0-50%：早期
- 50-200%：验证期
- 200-500%：拥挤期
- 500%+：高警惕

例外：

如果财报、订单、backlog、毛利或指引同步上修，可以降低拥挤警报等级。

反常识警报：

```text
如果基本面符合要求
且已有需求验证
但需求验证后 1 个月涨幅低于 1 倍
则默认提高风险等级
```

这通常说明市场看到了我们没有看到的东西，需要排查：客户真实性、订单质量、技术路线、稀释、政策许可、估值口径或竞争格局。

## 5. 基本面标准

基本面不是传统财务健康评分，而是判断这家公司是否处在时代浪潮中最好的价值链位置。

四个问题：

1. 行业是否是高增长行业，是否处在时代 Beta 中。不属于时代 Beta，直接排除。
2. 公司所处领域是否存在供需错配。需求越强、供给越难提供，越好。
3. 公司业务是否是该领域最容易快速反映到收入的环节。已验证的器件、材料、资源价格通常比新兴设备更快兑现。
4. 公司业务是否是该领域最有价值、最稀缺的部分。越像关键瓶颈或价值链控制点，基本面越好。

全满足，才是最高档基本面。

## 6. 风险标准

风险分为四档：

- 高危
- 危险
- 一般
- 低风险

风险基于每家公司的证伪逻辑，而不是统一止损线。

每家公司必须列出 3-4 个最核心证伪点。

如果都无法证伪，则为低风险。

如果出现一个核心证伪点，则至少提升到一般。

如果出现多个证伪点，或一个证伪点直接破坏 thesis，则进入危险或高危。

## 7. 情绪标准

情绪不是普通社交噪音，而是判断普通人是否已经知道并想买这只股票。

核心指标：

- X/Twitter 提及量
- Google 搜索热度
- Reddit / Stocktwits 讨论量
- 官方媒体报道
- 媒体标题是否开始强调“上涨数倍”

最高情绪档：

```text
普通人已经知道这只股票
官方媒体报道它涨了几倍
搜索热度暴涨
社交讨论从专业圈扩散到大众圈
```

如果情绪进入最高档，而订单、backlog、收入、毛利、指引没有同步上修，触发卖出警报。

## 8. 综合性价比

权重优先级：

```text
需求验证 = 基本面 > 性价比 > 风险
```

性价比只讨论时间和赔率。

时间来自：需求验证到收入兑现的周期。

赔率来自：可比公司分析和未来乐观估值情景。

风险是扣分项，但不能替代需求验证和基本面判断。

## 9. 不可容忍风险

每家公司都必须有专属止损条件。

通用红线：

- 需求证伪
- pipeline 不转 backlog
- backlog 不转 revenue
- 核心客户取消或认证延迟
- 技术路线被绕开
- 供给瓶颈被解决
- 大额稀释
- 政策许可失败
- 媒体热度暴涨但基本面没有同步上修

触发红线后，不讨论信仰，只讨论减仓、止损或移出观察池。

## 10. 组合与技术面执行

组合不是一次性满仓，而是把“投研正确”和“买点正确”分开处理。

组合原则：

- 总资金按 100% 管理，先定义目标仓位，再定义首批模型仓。
- 首批仓位只买已经满足需求验证和基本面的部分，剩余资金等待技术面与公司验证信号。
- 技术面只负责确认节奏，不能推翻需求证伪。
- 强趋势票不在单日长阳追满仓，只在回踩、突破后回踩或基本面继续上修时加仓。
- 高 beta 票必须分批，第一笔用于观察市场是否承认 thesis。

技术面确认指标：

- 20D / 50D：判断趋势和回踩支撑。
- RSI 14：判断是否进入低吸区或过热区。
- 20D 量能比：判断突破是否有资金确认。
- 收盘位置：是否重新站回 5D/20D，避免盘中假反弹。

买入框架：

```text
首批买入 = 投研入池 + 风险未触发 + 技术面不在明显追高区
加仓买入 = 需求验证继续增强 + 价格回踩不破或突破后回踩确认
满目标仓 = pipeline/backlog/订单/收入至少一个指标继续上修
```

卖出框架：

```text
减仓 = 技术破位 + 验证指标没有同步上修
止损 = 公司专属证伪点触发
清仓 = 需求被证伪或供需错配逻辑消失
```

## 11. 动作状态

状态必须直接服务执行，只使用四档：

- 加仓：需求验证、基本面和剩余赔率仍匹配，风险和情绪没有压过新增赔率，技术面不在明显追高区。
- 持仓：核心逻辑仍在，但价格、热度、验证节奏或风险不足以支持主动加仓。
- 减仓：风险、媒体热度、验证后涨幅或技术延伸已经压过新增赔率，但 thesis 尚未完全证伪。
- 清仓：需求被证伪、供需错配逻辑消失、公司专属红线触发，或风险已经破坏原始 thesis。

系统不使用模糊状态替代动作。观察、可跟踪、关注只能作为备注，不能作为最终动作。

## 12. 复盘系统

复盘固定每周进行一次，财报和交易发生时追加事件复盘。

必须保留四类复盘：

- 每周复盘：组合动作变化、风险灯变化、热度卖点、验证后涨幅、下周动作。
- 财报复盘：pipeline、backlog、订单、收入、毛利、现金流、指引是否继续验证需求。
- 交易复盘：买入、加仓、减仓、清仓的价格、理由、技术面状态和后续结果。
- 错误归因：逻辑错、时间错、赔率错、风险漏识别、情绪卖点没执行、数据错。

每次复盘都必须输出可执行结果，而不是只写观点。

```text
复盘输出 = 下周动作 + 证伪点变化 + 规则修订 + 新股入池/淘汰
```

## 13. 全球筛选范围

工业瓶颈是全球供应链问题，股票池不限定美股。

系统必须支持：

- 美国、加拿大、欧洲、日本、韩国、台湾、香港、澳洲等主要市场。
- 本地股、ADR、双重上市和主上市映射。
- 不同交易所代码格式，例如 SIVE.ST、XFAB.PA。
- 汇率统一换算为美元。

核心可比字段统一使用美元口径：

- 市值
- 收入
- 净利润 / EBITDA / 经营利润
- 现金与债务
- 订单 / backlog / pipeline
- 自由现金流
- 融资和摊薄

如果美元口径无法核验，该股票只能进入待核验池，不能直接进入组合候选。

## 14. 每周新股入池流水线

每周扫描一次全球候选股。

流水线：

```text
发现池 -> 待核验 -> 验证入池 -> 组合候选 -> 淘汰池
```

前端必须至少保留三个列表：

- 自选股票：用户正在重点跟踪和组合评估的股票。
- 发现池：每周扫描发现的全球候选股，只能说明为什么值得看，不能直接算验证后涨幅。
- 验证池：已经有初步订单、backlog、客户认证、财报或政策许可线索，等待人工确认是否进入自选。

发现池和验证池的股票必须允许手动加入自选；加入后才进入正式评分、动作状态和组合候选讨论。

进入验证入池必须同时具备：

- 股票 + 交易所
- 国家、交易所、本币、美元换算口径
- 行业/领域
- 瓶颈位置
- 时代 Beta
- 可追踪指标
- 验证入池日
- 验证价格
- 验证触发事件
- 核心证伪点

纯概念、无追踪指标、无法换算美元核心字段、无法说明验证事件的股票，不能进入验证入池。
